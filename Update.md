# Update

## 2026-05-31

### 已完成

- `perf:` 重复弹幕过滤从列表线性扫描改为队列加计数表，查重为平均 O(1)，窗口回收仍按用户设置的最近 N 条限制，主 App 和 TV 端逻辑一致。

- `update:` Android / iOS 全屏长按快捷入口支持自定义启用项和顺序，默认关注列表、观看历史、同类推荐全部展示；只启用一个时长按直接打开，全部关闭时提示“没有东西可展示”。

- `fix:` 播放页临时静音规则调整为：静音后上滑/下滑调节音量和点击静音按钮可解除；系统音量键变化不解除当前直播间静音，避免多开时误把已静音房间恢复声音。
- `update:` 实时字幕模型路径改为选择“关键 onnx 文件”，App 会自动识别同目录模型类型并校验配套文件；例如中级模型选择 `model.int8.onnx`，缺少 `tokens.txt`、`config.yaml`、`am.mvn` 时会直接提示缺失项。
- `fix:` 字幕模型路径展示不再为了显示完整路径撑长 UI，只显示关键文件名，说明行最多两行并显示模型识别或缺失信息。
- `update:` 实时字幕样式补齐字号、可拖动/百分比位置、颜色、粗细、背景开关；默认仍在下方附近，播放页内可直接调整。
- `update:` 实时字幕补充位置锁定；锁定后播放页只显示字幕，鼠标悬停到字幕区域才显示解锁按钮，移动端可长按解锁。
- `update:` 实时字幕桌面端接入本地 ASR 实验链路：播放直播时用第二个 `media_kit` 解码器跟随当前播放地址，将音频导出为 16k PCM 后喂给 `sherpa_onnx`；支持 Paraformer、Whisper、Zipformer 推荐模型。
- `fix:` 实时字幕启动失败会自动关闭开关，并加入启动保护标记；如果上次 ASR 初始化导致进程退出，下次启动会自动禁用实时字幕，避免应用反复闪退无法进入。
- `fix:` Windows 从最大化窗口双击进入全屏时，先恢复成普通居中窗口，再进入全屏，避免最大化窗口边框状态残留导致右下偏移和空边。
- `fix:` 抖音账号入口按平台调整：手机端改为打开抖音 App，并提示 Cookie 需要从桌面浏览器获取完整 Request Headers；桌面端继续打开系统浏览器。
- `fix:` 抖音 Cookie 登录框兼容纯 Cookie、`Cookie: xxx` 和整段请求头粘贴；如果浏览器复制出来是两列格式里的 `cookie` + 下一行值，App 会自动提取下一行的完整 Cookie。
- `verify:` 已用桌面浏览器登录后的抖音请求头 Cookie 验证 `aweme/v1/web/live/search/`，返回 `status_code: 0` 且有直播搜索数据；用户只填 `ttwid` 不足以保证主播 / 房间搜索可用。
- `fix:` 搜索页默认平台记忆：从首页当前平台进入搜索时默认选中该平台，并保留上一次搜索页停留的平台。
